调用链（Trace）是一个有向无环图（DAG），其顶点叫做跨度（Span），边叫做Reference。每个Span包含

  • operation name
  • start timestamp(local ?)
  • finish timestamp(local ?)
  • span tag, array of k-v pairs
  • span log, array of span log
  • span context (optional)
  • references (optional)

zipkin是Twitter基于Google的Dapper论文构建的OpenTracing系统。在zipkin中，Span包含

  • 全局唯一的调用链编号
  • 调用链内唯一的span编号
  • 父span编号
  • 开始时间
  • 持续时间
  • 附加数据（span tag）

在运行时，ZipKin分为ZipKin Server和ZipKin Client。

ZipKin Client作为库嵌入到调用服务的程序中。当程序调用服务时，ZipKin Client将调用的服务名、耗时等信息上传给ZipKin Server。ZipKin Server复制收集、维护和展示调用链信息。ZipKin Server又分为4个部分

  • Collector。负责收集数据。
  • Storage。负责存储数据。支持内存、MySQL、ElasticSearch等。
  • API。负责查询存储的数据。
  • Web。提供Web管理界面。

ZipKin Server的命令行参数见 zipkin-server README

在ZipKin Client的go客户端（zipkin-go）中，如果使用http上传跨度，上传过程是异步的。代码如下 

// https://github.com/openzipkin/zipkin-go/blob/master/reporter/http/http.go
func (r *httpReporter) loop() {
        var (
                nextSend   = time.Now().Add(r.batchInterval)
                ticker     = time.NewTicker(r.batchInterval / 10)
                tickerChan = ticker.C
        )
        defer ticker.Stop()

        for {
                select {
                case span := <-r.spanC:
                        currentBatchSize := r.append(span)
                        if currentBatchSize >= r.batchSize {
                                nextSend = time.Now().Add(r.batchInterval)
                                go func() {
                                        _ = r.sendBatch()
                                }()
                        }
                case <-tickerChan:
                        if time.Now().After(nextSend) {
                                nextSend = time.Now().Add(r.batchInterval)
                                go func() {
                                        _ = r.sendBatch()
                                }()
                        }
                case <-r.quit:
                        r.shutdown <- r.sendBatch()
                        return
                }
        }
}

如果使用PHP客户端，消息发送是同步的。 

// https://github.com/openzipkin/zipkin-php/blob/master/src/Zipkin/Reporters/Http.php
public function report(array $spans): void
{
    $payload = json_encode(array_map(function (Span $span) {
        return $span->toArray();
    }, $spans));
    if ($payload === false) {
        $this->logger->error(
            sprintf('failed to encode spans with code %d', \json_last_error())
        );
        return;
    }
    $client = $this->clientFactory->build($this->options);
    try {
        $client($payload); // 实际上调用的是curl_exec，见 https://github.com/openzipkin/zipkin-php/blob/master/src/Zipkin/Reporters/Http/CurlFactory.php
    } catch (RuntimeException $e) {
        $this->logger->error(sprintf('failed to report spans: %s', $e->getMessage()));
    }
}

在Java里可以使用zipkin2.reporter.AsyncReporter进行异步信息上传。

如果使用gPRC通信，可以实现io.grpc.ClientInterceptor或io.grpc.ServerInterceptor，在不改变应用层代码的情况下上传调用链信息。 

import org.apache.logging.log4j.LogManager;
import org.apache.logging.log4j.Logger;

import zipkin2.Span;
import zipkin2.reporter.AsyncReporter;
import zipkin2.reporter.okhttp3.OkHttpSender;

public class TraceAgent {
    private static final Logger log = LogManager.getLogger(TraceAgent.class);
    private static TraceAgent instance;
    private AsyncReporter<zipkin2.Span> report;

    // 这里没有保证report != null。
    private TraceAgent(String ip) {
        try {
            OkHttpSender sender = OkHttpSender.create("http://" + ip);
            report = AsyncReporter.builder(sender).build();
        } catch (Exception e) {
            e.printStackTrace();
            log.error("error of opentracing initialization,It may be that hosts is not configured!");
        }
    }

    // 注意，此处返回的对象可能是Null
    public static TraceAgent getInstance(String ip) {
        try {
            if (instance == null) {
                synchronized (TraceAgent.class) {
                    if (instance == null) {
                        instance = new TraceAgent(ip);
                    }
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
            log.error("error of opentracing initialization,It may be that hosts is not configured!");
        }
        return instance;
    }

    // 如果report初始化失败，可能出现调用链信息无法上传。这时应当改用本地日志记录调用链信息。以便后续手动导入ZipKin Server。
    public void send(final Span span) {
        try {
            if (report != null) {
                report.report(span);
                report.flush();
            }
        } catch (Exception e) {
            log.error("report zipkin error :" + e.getMessage(), e);
        }

    }
}

在Python中使用zipkin时，需要安装zipkin或py-zipkin包。 

# https://github.com/Yelp/py_zipkin
import random
import requests
import time

from py_zipkin.zipkin import zipkin_span
from py_zipkin.transport import BaseTransportHandler

class HttpTransport(BaseTransportHandler):

    def get_max_payload_bytes(self):
        return None

    def send(self, encoded_span):
        # The collector expects a thrift-encoded list of spans.
        requests.post(
            'http://localhost:9411/api/v1/spans',
            data=encoded_span,
            headers={'Content-Type': 'application/x-thrift'},
        )


def do_stuff(a, b):
    time.sleep(random.random())

def some_function(a, b):
    with zipkin_span(
            service_name='my_service',
            span_name='my_span_name',
            transport_handler=HttpTransport(),
            port=9411,
            sample_rate=100, #0.05, # Value between 0.0 and 100.0
    ):
        do_stuff(a, b)

for x in range(0, 100):        
    some_function(1, 2)
