Spring实战笔记

Spring简介

Spring是一个以依赖注入为核心的Java开发框架。Spring的优点有：

功能强大。Spring提供了大量的可重用组件，也提供了对大量中间件的封装。 开发简单。通过依赖注入将业务逻辑和基础服务解耦，让开发者专注于业务逻辑。 提供Spring MVC和Spring Webflux，大大简化RESTful服务的开发。 提供Spring Boot，简化Spring项目的创建。

Spring的核心是控制反转和依赖注入。我们先看一个例子： 

package com.springinaction.knights.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import com.springinaction.knights.BraveKnight;
import com.springinaction.knights.Knight;
import com.springinaction.knights.Quest;
import com.springinaction.knights.SlayDragonQuest;

@Configuration
public class KnightConfig {

    @Bean
    Knight knight() {
        return new BraveKnight(quest());
    }

    @Bean
    public Quest quest() {
        return new SlayDragonQuest(System.out);
    }
}

然后通过ApplicationContext得到注入的对象： 

package com.springinaction.knights;

import org.springframework.context.support.ClassPathXmlApplicationContext;

public class KnightMain {

    public static void main(String[] args) throws Exception {
        ClassPathXmlApplicationContext context = new ClassPathXmlApplicationContext("META-INF/spring/knights.xml");
        Knight knight = context.getBean(Knight.class);
        knight.embarkOnQuest();
        context.close();
    }
}

这些上下文对象的构建非常简单： 

new FileSystemXmlApplicationContext("c:/knight.xml");
new ClassPathXmlApplicationContext("knight.xml");
new AnnotationConfigApplicationContext(com.springinaction.knights.config.KnightConfig.class);

Bean

在Spring中，用来进行注入的对象叫做bean。Bean可以是任意类型的对象。Spring负责创建bean，并维护bean的生命周期：

Spring对bean进行实例化。 Spring使用值和引用对bean注入属性。 如果bean实现了BeanNameAware接口，Spring将bean的id传递给setBeanName()方法。 如果bean实现了BeanFactoryAware接口，Spring将调用setBeanFactory()方法，将BeanFactory实例传入。 如果bean实现了ApplicationContextAware接口，Spring将调用setApplicationContext()方法，传入上下文。 如果bean实现了BeanPostProcessor接口，Spring将调用postProcessBeforeInitialization()方法。 如果bean实现了InitializingBean接口，Spring将调用afterPropertiesSet()方法。如果bean使用<init-method>标签声明了初始化方法，也会被调用。 如果bean实现了BeanPostProcessor接口，Spring将调用postProcessAfterInitialization()方法。 将bean注入到目标对象中，提供服务，直到上下文被销毁。 在上下文销毁时，如果bean实现了DisposableBean接口，Spring将调用destroy()方法。如果通过<destroy-method>标签声明了销毁方法，Spring也会调用。

将bean注入到目标对象的操作叫做装配。Spring提供了3中bean装配方案：通过XML装配、通过Java装配和自动装配。这里只介绍自动装配。要完成自动装配，Spring必须知道

  1. 哪些对象需要被注入？
  2. 哪些对象可以作为bean注入到目标对象？

为了得到答案，Spring需要进行组件扫描（scan component）。而要使组件扫描正常工作，开发者需要：

  1. 将需要被注入的对象声明为@Autowired。
  2. 将bean的类声明以下注解之一：@Bean、@Component、@Controller、@Service或@Repository。
  3. 在Spring配置类中声明@ComponentScan，启动组件扫描。

我们通过例子一步一步来了解。首先，将被注入对象声明为@Autowired： 

import org.springframework.beans.factory.annotation.Autowired;

public class CDPlayer implements MediaPlayer {
    @Autowired
    private CompactDisc cd;

    public void play() {
        cd.play();
    }
}

然后将bean的类声明为@Bean： 

import org.springframework.context.annotation.Bean;

@Bean
public class SgtPeppers implements CompactDisc {
    // ...
}

最后在配置中开启扫描： 

import org.springframework.context.annotation.ComponentScan;
improt org.springframework.context.annotation.Configuration;

@Configuration
@ComponentScan
public class CDPlayerConfig {
}

bean可以具有名字： 

@Component("lonelyHeartsClub")
public class SgtPeppers implements CompactDisc {
  // ...
}

之后就可以让Spring进行自动装配：

@Autowired也可以用在任意函数上： 

@Autowired
public void insertDisc(CompactDisc cd) {
    this.cd = cd;
}

使用Java也可以进行装配。这么做需要建立一个配置类，在其中声明bean，并手动装配： 

import org.springframework.context.annotation.Configuration;

@Configuration
public class CDPlayerConfig {
    @Bean
    public CompactDisc sgtPeppers() {
        return new SgtPeppers();
    }

    @Bean
    public CDPlayer cdPlayer() {
        return new CDPlayer(sgtPeppers()); // 手动装配
    }
}

除了自动装配和Java装配外，还可以使用传统的XML方式进行装配。

为了支持在不同环境（开发环境、生产环境）下进行自动装配，Spring提供了注解@Profile进行按环境装配。@Profile可以应用于类或方法上： 

@Configuration
@Profile("prod")
public class ProductionProfileConfig {
    // ....
}

Configuration
public class Config {
    @Bean
    @Profile("dev")
    public DataSource embeddedDataSource() { ... }
}

如果要在XML中指定环境，可以在<beans>标签中添加属性profile： 

<beans profile="dev" ...>
   ...
</beans>

使用XML时要注意，<beans>标签是可以嵌套的。

Spring提供了两个参数选择要激活的环境，按照有限次序，分别是：spring.profiles.active和spring.profiles.default。Spring也提供了多种参数设置方式：

使用DispatcherServlet初始化参数。 使用ApplicationContext参数。 使用环境变量。 使用JVM系统属性。 使用@ActiveProfiles注解。

为了处理装配过程中发生的歧义性问题，Spring提供了@Primary注解，用于指定首选装配方法： 

@Bean
@Primary
public Dessert iceCream() { ... }

对于的XML是： 

<bean primary="true" ... >

bean作用域

Spring为bean定义了多种作用域：

单例（singleton）。在整个应用中只有一个bean实例。 原型（prototype）。每次注入或通过上下文获取时，都创建一个新的bean实例。 会话（session）。在Web应用中为每个会话建立一个bean实例。 请求（request）。在Web应用中为每个请求建立一个bean实例。

默认情况下，bean采用单例作用域。Spring提供了@Scope来设置bean的作用域： 

@Bean
@Scope('prototype')
public class Notepad {...}

Spring提供了注解@PropertySource，用于将环境变量注入Environment对象： 

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.PropertySource;
import org.springframework.core.env.Environment;

@Configuration
@PropertySource("classpath:/com/soundsystem/app.properties")
public class Config {
    @Autowired
    Environment env;

    @Bean
    public BlankDisc disc() {
        return new BlankDisc(env.getProperty("disc.title"), env.getProperty("disc.artist"));
    }
}

Spring中的面向切面编程

Spring支持5中类型的通知（advice）：

前置通知（Before）。 后置通知（after）。 返回通知（after-returning）。 异常通知（after-throwing）。 环绕通知（around）。等效于before+after。

Spring是通过代理对象完成织入（weaving）的。Spring使用AspectJ的切点表达式：

arg() 参数 @args() execution() 方法 this() target @target() within() @within() @annotation

在使用AOP时，首先要定义切面，然后将切面声明为bean，最后让Spring启用AOP。定义切面的方法如下： 

import org.aspectj.lang.annotation.AfterReturning;
import org.aspectj.lang.annotation.AfterThrowing;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;

@Aspect
public class Audience {
    @Pointcut("execution(** concert.Performace.perform(..))")
    public void performance() {}

    @Before("performance()")
    public void silenceCellPhones() {
        system.out.println("Silencing cell phones");
    }

    @Before("performance()")
    public void takeSeats() {
        System.out.println("Taking seats");
    }

    @AfterReturning("performance()")
    public void applause() {
        System.out.println("CLAP CLAP CLAP!!!");
    }

    @AfterThrowing("performance()")
    public void silenceCellPhones() {
        System.out.println("Demanding a refund");
    }
}

接着装配bean： 

@Bean
public Audience audience() {
    return new Audience();
}

接着是启用AOP： 

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.EnableAspectJAutoProxy;

@Cofinguration
@EnableAspectJAutoProxy
@ComponentScan
public class ConcertConfig {
    @Bean
    public Audience audience() {
        return new Audience();
    }
}

Spring对Web的支持

Spring提供了Spring MVC作为Web开发框架之一（还有WebFlux）。Spring MVC是一个单一入口框架，入口DispatcherServlet。DispatcherServlet负责将请求转发到对应的组件，它是Spring MVC的核心。传统的配置保存在web.xml中。在Spring MVC中，也可以通过Java完成： 

import org.springframework.web.servlet.support.AbstractAnnotationConfigDispatcherServletInitializer;

public class MyWebAppInitializer extends AbstractAnnotationConfigDispatcherServletInitializer {
    @Override
    protected String[] getServletMappings() {
        return new String[]{"/"};
    }

    @Override
    protected Class<?>[] getRootConfigClasses() {
        return new Class<?>[]{RootConfig.class};
    }

    @Override
    protected Class<?>[] getServletConfigClasses() {
        return new Class<?>[]{WebConfig.class};
    }
}

这段代码的运行原理如下：

在启动时，Spring会寻找javax.servlet.ServletContainerInitializer的实现，用来配置Servlet容器。Spring自己提供了实现SpringServletContainerInitializer，后者会查找WebApplicationInitializer的实现，并委派任务。AbstractAnnotationConfigDispatcherServletInitializer是WebApplicationInitializer的派生类。

在Spring MVC中，控制器只需要加上@Controller注解： 

import org.springframework.stereotype.Controller;

        return "home"; // 视图名字。

异常

异常可以使用@ResponseStatus映射状态码。方法可以添加@ExceptionHandler处理异常。 

@ExceptionHandler(DuplicateException.class)
public String handleException() {
    return "erro/duplicate";
}

这些异常处理函数可以集中到一个类中，这个类需要以@ControllerAdvice声明。 

@ControllerAdvice
public class ExceptionHandler {
    @ExceptionHandler(MyException.class)
    public String handleException() {
        return "error/duplicate";
    }
}
