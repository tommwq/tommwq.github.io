入门

Spring组件是轻量的，但配置却是重量级的。一开始是XML配置，2.5开始引入了注解配置，3.0引入基于Java的配置。但是在使用某些Spring特性，或使用第三方库时，仍然无法避免手动配置。 Spring Boot的一大特性就是支持自动配置。此外，Spring Boot的重要特性还有starter、Spring Boot CLI和Actuator。

Spring Boot CLI是一个用于简化Spring程序开发的命令行工具，它可以从Spring Boot的网站（从 https://docs.spring.io/spring-boot/docs/current/reference/html/getting-started-installing-spring-boot.html 可以找到Spring Boot CLI的最新版本下载地址）上下载。安装Spring Boot CLI之后，可以在系统中使用spring命令来创建或运行Sping程序。比如下面的命令会建立一个spring项目，项目使用web、jpa和security组件，使用gradle构建，打包成jar，并保存到目录myapp中： 

spring init --dependencies web,jpa,security --build gradle --packaging jar --extract myapp

spring init常用的参数有：

参数 说明 默认值
–list 列出参数  
artifactId 构件名 demo
groupId 组名 com.example
version 版本 0.0.1-SNAPSHOT
packageName 包名 com.example.demo
packaging 打包方式 jar
name 项目名 demo

开发第一个应用程序

Spring Boot程序的入口类以@SpringBootApplication注解。它是以下三个注解的组合：

  • @Configuration 表明该类使用Spring基于Java的配置。
  • @ComponentScan 启用组件扫描，自动发现并注册Bean。
  • @EnableAutoConfiguration 启用Spring Boot自动配置。

使用Spring Boot CLI创建的项目，可以用gradle bootRun运行，或者使用gradle build构件后，使用java -jar build/libs/Project-0.0.1-SNAPSHOT.jar运行。

Spring Boot程序包含spring-boot-autoconfiguration包，其中有很多配置类。这些配置类在一定条件下才会激活。这就是Spring 4.0引入的条件化配置机制。下面是一个条件化配置的例子： 

import org.springframework.context.annotation.Condition;
import org.springframework.context.annotation.ConditionContext;
import org.springframework.core.type.AnnotatedTypeMetadata;

public class JdbcTemplateCondition implements Condition {
  @Override
  public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
    try {
      context.getClassLoader().loadClass("org.springframework.jdbc.core.jdbcTemplate");
      return true;
    } catch (Exception e) {
      return false;
    }
  }
}

@Conditional(JdbcTemplateCondition.class)
public MyService myService() {
  // ...
}

在上面的例子中，只有classpath中存在JdbcTemplate时，MyService Bean才会创建。Spring Boot提供了以下的条件化注解：

@ConditionalOnBean 配置了特定Bean
MissiongBean 没有配置特定Bean
@ConditionalOnClass classpath包含指定类
@ConditionalOnMissingClass classpath缺少指定类
@ConditionalOnExpression Spring Expression Language表达式结果为true
@ConditionalOnJava 某个Java版本或版本范围
@ConditionalOnJndi  
@ConditionalOnProperty 属性具有指定值
@ConditionalOnResource classpath中包含指定资源
@ConditionalOnWebApplication 应用是Web应用
@ConditionalOnNotWebApplication 应用不是Web应用
