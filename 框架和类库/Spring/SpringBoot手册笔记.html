起步

Spring Boot要求Java 8以上，要求使用Maven 3.3+或Gradle 4.4+。对于Servlet容器，Spring Boot支持Tomcat 9.0+、Jetty 9.4+和Undertow 2.0+。

下面是一个使用Spring Boot的项目POM框架 

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
        <modelVersion>4.0.0</modelVersion>
        <groupId>com.example</groupId>
        <artifactId>myproject</artifactId>
        <version>0.0.1-SNAPSHOT</version>
        <!-- Inherit defaults from Spring Boot -->
        <parent>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-starter-parent</artifactId>
                <version>2.1.1.RELEASE</version>
        </parent>
        <!-- Add typical dependencies for a web application -->
        <dependencies>
                <dependency>
                        <groupId>org.springframework.boot</groupId>
                        <artifactId>spring-boot-starter-web</artifactId>
                </dependency>
        </dependencies>
        <!-- Package as an executable jar -->
        <build>
                <plugins>
                        <plugin>
                                <groupId>org.springframework.boot</groupId>
                                <artifactId>spring-boot-maven-plugin</artifactId>
                        </plugin>
                </plugins>
        </build>
</project>

下面展示如何使用Spring Boot编写一个Web服务。首先根据上面的框架建立POM。之后加入spring-boot-starter-web依赖项::

<dependencies> … <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-web</artifactId> </dependency> </dpendencies>

接着建立src/main/java目录和src/main/java/Example.java文件，编写代码::

import org.springframework.boot.*; import org.springframework.boot.autoconfigure.*; import org.springframework.web.bind.annotation.*;

@RestController @EnableAutoConfiguration public class Example {

@RequestMapping("/") String home() { return "Hello World!"; }

public static void main(String[] args) throws Exception { SpringApplication.run(Exaple.class, args); } }

这个文件里使用了几个注解。第一个是@RestController。这是一个模板注解（stereotype annotation），告诉Spring我们编写的类可以处理Web请求。@RequestMapping注解提供了路由信息，告诉Spring将路径"/"映射到home方法。@RestController同时告诉Spring直接返回字符串作为渲染结果。第二个类注解是@EnableAutoConfiguration，它告诉Spring根据依赖项自动加载配置。spring-boot-starter-web会引用Tomcat和SpringMVC，因此Spring知道你在开发一个Web程序。最后就是方法SpringApplication.run。这个方法告诉Spring启动程序。

现在我们看看这个程序的运行效果::

mvn spring-boot:run

打开浏览器，访问http://localhost:8080，可以看到::

Hello World!

现在按Ctrl-C退出程序，然后将程序打包::

mvn package

这个命令会在target目录下生成myproject-0.0.1-SNAPSHOT.jar。现在测试一下我们的jar包::

java -jar target/myproject-0.0.1-SNAPSHOT.jar

使用Spring Boot ==========

Spring Boot提供了一组starter来简化应用构建，它们是：

应用启动器（application starter） spring-boot-starter。核心starter，包括自动配置支持、日志和YAML。 spring-boot-starter-activemq。通过ActiveMQ使用JMS消息。 spring-boot-starter-amqp。Spring AMAQ和Rabbit MQ。 spring-boot-starter-aop。Spring AOP和AspectJ。 spring-boot-starter-artemis。通过Artemis使用JMS消息。 spring-boot-starter-batch。Spring Batch。 spring-boot-starter-cache。Spring框架缓存。 spring-boot-starter-cloud-connectors。Spring Cloud Connectors。 spring-boot-starter-data-cassandra。Cassandra分布式数据库。 spring-boot-starter-data-cassandra-reactive。Cassandra分布式数据库。 spring-boot-starter-data-couchbase。Couchbase数据库。 spring-boot-starter-data-couchbase-reactive。Couchbase数据库。 spring-boot-starter-data-elasticsearch。Elastic search。 spring-boot-starter-data-jdbc。Spring Data JDBC。 spring-boot-starter-data-jpa。Hibernate。 spring-boot-starter-data-ldap。LDAP。 spring-boot-starter-data-mongodb。MongoDB。 spring-boot-starter-data-mongodb-reactive。MongoDB。 spring-boot-starter-data-eno4j。Neo4J，一个图数据库。 spring-boot-starter-data-redis。Redis。 spring-boot-starter-data-redis-reactive。 spring-boot-starter-data-rest。RESTful资源。 spring-boot-starter-data-solr。Solr。 spring-boot-starter-freemarker。FreeMarker视图。 spring-boot-starter-groovy-templates。Groovy Templates视图。 spring-boot-starter-hateoas。HATEOAS。 spring-boot-starter-integration。 spring-boot-starter-jdbc。HikariCP连接池。 spring-boot-starter-jersey。使用JAX-RS和Jersey构建Web应用。 spring-boot-starter-jooq。通过JOOQ访问数据库。 spring-boot-starter-json。读写JSON。 spring-boot-starter-jta-atomikos。 spring-boot-starter-jta-bitronix。 spring-boot-starter-mail。 spring-boot-starter-mustache。 spring-boot-starter-oauth2-client。 spring-boot-starter-oauth2-resource-server。 spring-boot-starter-quartz。调度器。 spring-boot-starter-security。 spring-boot-starter-test。JUnit、Hamcrest和Mockito。 spring-boot-starter-thymeleaf。 spring-boot-starter-validation。 spring-boot-starter-web。使用Tomcat和Spring MVC构建RESTful服务。 spring-boot-starter-web-service。 spring-boot-starter-webflux。 spring-boot-starter-websocket。 spring-boot-starter-actuator。

技术启动器（technical starter） spring-boot-starter-jetty。Jetty容器。 spring-boot-starter-log4j2。Log4j2日志。 spring-boot-starter-logging。Logback日志（默认）。 spring-boot-starter-reactor-netty。Reactor Netty容器。 spring-boot-starter-tomcat。Tomcat容器。 spring-boot-starter-undertow。Undertow容器。

spring-boot-start-web是比较常用的，它实际上是下列构件的组合::

org.springframework.boot:spring-boot-starter org.springframework.boot:spring-boot-starter-json org.springframework.boot:spring-boot-starter-tomcat org.hibernate.validator:hibernate-validator org.springframework:spring-web org.springframework:spring-webmvc

如果将应用类（application class）置于根目录，并且为应用类增加注释@ComponentScan，其他目录下的组件（@Component、@Service、@Repository、@Controller）会自动注册为Spring Bean。Spring boot提供了一个简化的注解@SpringBootApplication。使用@SpringBootApplication相当于使用::

@EnableAutoConfiguration @ComponentScan @Configuration

Spring Boot提供了调试工具spring-boot-devtools::

<dependencies> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-devtools</artifactId> <optional>true</optional> </dependency> </dependencies>

configurations { developmentOnly runtimeClasspath { extendsFrom developmentOnly } }

dependencies { developmentOnly("org.springframework.boot:spring-boot-devtools") }

Spring Boot通常用于构建Web应用。但是它也支持控制台程序::

package hello;

import java.util.Arrays;

import org.springframework.boot.CommandLineRunner; import org.springframework.boot.SpringApplication; import org.springframework.boot.autoconfigure.SpringBootApplication; import org.springframework.context.ApplicationContext; import org.springframework.context.annotation.Bean;

@SpringBootApplication public class Application {

public static void main(String[] args) { SpringApplication.run(Application.class, args); }

@Bean public CommandLineRunner commandLineRunner(ApplicationContext ctx) { return args -> {

System.out.println("Let's inspect the beans provided by Spring Boot:");

String[] beanNames = ctx.getBeanDefinitionNames(); Arrays.sort(beanNames); for (String beanName : beanNames) { System.out.println(beanName); }

}; }

}

Spring Boot特性 ==========

Spring Boot提供了若干FailureAnalyzer，在应用启动失败后，FailureAnalyzer会提供检查建议。如果想看详细信息，可以使用–debug参数启动应用::

java -jar myproject-0.0.1.SNAPSHOT.jar –debug

Spring Boot在启动时会显示banner。banner可以自行设定，方法是修改属性spring.banner.location，或者在classpath下建立banner.txt文件。如果banner.txt没有使用UTF-8编码，需要设置spring.banner.charset告诉Spring Boot如何读取这个文件。banner.txt里可以包含一些占位符：

${application.version}。MANIFEST.MF中声明的版本号，比如“Implementation-Version: 1.0”显示为“1.0”。 ${application.formatted-version}。以(v1.0)格式显示版本号。 ${spring-boot.version}。Spring Boot版本。 ${spring-boot.formatted-version}。 ${application.title}。MANIFEST.MF中声明的标题。“Implementation-Title: MyApp”显示为“MyApp”。

属性spring.main.banner-mode可以控制将banner输出到控制台（console）、日志（log）或不显示（off）。也可以使用代码来控制::

public static void main(String[] arg) { SpringApplication app = new SpringApplication(MySpringConfiguration.class); app.setBannerMode(Banner.Mode.OFF); app.run(args); }

SpringApplication会建立合适的ApplicationContext对象。如果使用Spring MVC，会建立AnnotationConfigServletWebServerApplicationContext。如果使用Spring WebFlux，会建立AnnotationConfigReactiveWebServiceApplicationContext。其他情况会建立AnnotationConfigApplicationContext。

Bean可以访问程序启动参数。方法是在构造函数中增加org.springframework.boot.ApplicationArguments型参数::

@Component public class MyBean { @Autowired public MyBean(ApplicationArguments args) { boolean debug = args.containsOption("debug"); // … } }

如果有代码需要在SpringApplication.run()之前执行一次，可以实现ApplicationRunner或CommandLineRunner接口。这两个接口只有一个void run(String… args)方法。

Spring Boot从下列位置加载application.properties文件：

当前目录的config子目录。 当前目录。 classpath的/config包。 classpath root。

使用命令行参数spring.config.name和spring.config.location可以改变配置名和配置路径::

java -jar myproject.jar –spring.config.name=myproject –spring.config.location=classpath:/default.properties,classpath:/override.properties

配置中可以使用占位符，占位符会根据环境变量填充::

app.name=MyApp app.description=${app.name} is Spring Boot application

Spring Profile提供了对特定场景配置的支持。@Profile需要应用在@Component和@Configuration注解的类上::

@Configuration @Profile("production") public class ProductionConfigruation { // … }

环境spring.profiles.active可以控制活跃Profile::

spring.profiles.active=dev,hsqldb

spring.profiles=prod spring.profiles.include=proddb,prodmq

Spring Boot提供了属性logging.file和loggin.path来指定日志输出目标，日志基本由logging.level控制::

logging.level.root=WARN logging.level.org.springframework.web=DEBUG logging.level.org.hibernate=ERROR

日志也是可以分组，并按组设置级别::

logging.group.tomcat=org.apache.catalina, org.apache.coyote, org.apache.tomcat logging.level.tomcat=TRACE

Spring Boot预定义了两个日志分组::

web => org.springframework.core.codec, org.springframework.http, org.springframework.web sql => org.springframework.jdbc.core, org.hibernate.SQL

环境变量logging.config指定了日志配置文件。

Spring Boot提供了3个JSON库：GSON、Jackson和JSON-B。Jackson是默认的库。

在Web开发领域，Spring Boot提供了Spring MVC和webflux两个框架。下面介绍Spring Boot对Spring MVC的支持。Spring MVC使用HttpMessageConverter对HTTP请求和应答进行编码或解码，你可以自己实现它::

import org.springframework.boot.autoconfigure.web.HttpMessageConverters; import org.springframework.context.annotation.*; import org.springframework.http.converter.*; @Configuration public class MyConfiguration { @Bean public HttpMessageConverters customConverters() { HttpMessageConverter<?> additional = … HttpMessageConverter<?> another = … return new HttpMessageConverters(additional, another); } }

Spring Boot的静态文件保存在/static、/public、/resources或/META-INF/resources下（相对classpath或ServletContext根目录）。设置属性spring.resources.static-locations可以修改这个路径。错误页面（比如404）可以放在resources/public/error目录下::

resources/public/error/404.html

Spring MVC支持CORS（Cross-origin resource sharing）。开启CQRS需要定义一个WebMvcConfigurer对象::

@Configuration public class MyConfiguration { @Bean public WebMvcConfigurer corsConfigurer() { return new WebMvcConfigurer() { @Override public void addCorsMappings(CorsRegistry registry) { registry.addMapping("/api/**"); } }; } }

下面介绍WebFlux框架。WebFlus是Spring Framework 5.0新引入的反映式Web开发框架。WebFlux是异步的、非阻塞的，不依赖于Servlet API。WebFlux提供函数式和注解两种风格，注解的风格接近于Spring MVC::

@RestController @RequestMapping("/users") public class MyRestController {

@GetMapping("/{user}") public Mono<User> getUser(@PathVariable Long user) { // … }

@GetMapping("/{user}/customers") public Flux<Customer> getUserCustomers(@PathVariable Long user) { // … } }

函数式风格则不太一样::

@Configuration public class RoutingConfiguration { @Bean public RouterFunction<ServerResponse> monoRouterFunction(UserHandler userHandler) { return RouterFunctions.route(GET("/{user}").and(accept(APPLICATIONJSON)), userHandler::getUser) .addRoute(GET("/{user}/customers").and(accept(APPLICATIONJSON)), userHandler::getUserCustomers); } }

@Component public class UserHandler { public Mono<ServerResponse> getUser(ServerRequest request) { ServerResponse.ok().body(Mono.just("ok")); } public Mono<ServerResponse> getUserCustomers(ServerRequest request) { // … } }

如果要增加额外的WebFlux配置，可以定义WebFluxConfigurer型的@Configuration类。如果不想使用Spring Boot提供的默认配置，可以结合@Configuration和@EnableWebFlux。WebFlux使用HttpMessageReaders和HttpMessageWriters进行消息编码。WebFlux的静态文件和Spring MVC的默认路径一样，而配置名称不同::

spring.webflux.static-path-pattern=/resources/**

Spring Boot支持内置的Tomcat、Jetty或Undertow容器，以及Reactor Netty。

Spring Boot推荐使用HikariCP连接池。如果使用了spring-boot-starter-jdbc或spring-boot-starter-data-jpa，HikariCP会自动引入项目。数据源的配置如下::

spring.datasource.url=jdbc:mysql://localhost/test spring.datasource.username=root spring.datasource.password=123456 spring.datasource.driver-class-name=com.mysql.jdbc.Driver

Java Persistence API提供了Java对象和关系型数据库记录的映射方法。spring-boot-starter-data-jpa提供了：

Hibernate。一种流行的JPA实现。 Spring Data JPA。使开发基于JPA的仓库更容易。 Spring ORMs。Spring Framework核心ORM。

通常JPA实体类（Entity class）需要在persistance.xml中指定。在Spring Boot中，自动实体扫描可以替代这个文件。任何以@Entity注解的类都被注册为实体类::

import java.io.Serializable; import javax.persistence.*;

@Entity public class City implements Serializable { @Id @GeneratedValue private Long id;

@Column(nullable = false) private String name;

protected City() {} // JPA要求必须有无参数构造函数 public City(String name, String state) {…}

// getter and setter }

而仓库需要继承Repository或CrudRepository接口（不需要实现）::

import org.springframework.data.domain.*; import org.springframework.data.repository.*;

public interface CityRepository extends Repository<City,Long> { Page<City> findAll(Pageable pageable); City findByNameAndStateAllIgnoringCase(String name, String state); }

如果使用配置spring.jpa.hibernate.dll-auto=create-drop，可以自动创建数据库。

Spring为Redis提供了spring-boot-starter-data-redis。

使用@Cacheable注解可以增加缓存。使用spring-boot-starter-cache可以引入缓存依赖。缓存可以保存在Redis中。如果配置了Redis，一个RedisCacheManager会自动装载。

Spring Boot提供了RestTemplate类来调用远程RESTful服务::

RestTemplate restTemplate; restTemplate.getForObject("/{name}/details", Details.class, name);

如果配置了WebFlux，可以通过WebClient以反应式调用RESTful服务::

@Service public class MyService { private final WebClient webClient; public MyService(WebClient.Builder builder) { this.webClient = builder.baseUrl("http://example.org").build(); }

public Mono<Details> someRestCall(String name) { return this.webClient.get() .uri("{/{name}/details", name) .retrieve() .bodyToMono(Details.class); } }

Spring Boot提供了Quartz调度器::

spring-boot-starter-quartz

任务执行和调度。如果定义了注解@EnableAsync，Spring Boot会自动装配ThreadPoolTaskExecutor。Spring Boot提供了如下配置::

spring.task.execution.pool.max-threads=16 spring.task.execution.pool.queue-capacity=100 spring.task.execution.pool.keep-alive=10s

Spring会话::

spring.session.store-type=jdbc

spring-boot-starter-actuator提供了生产特性。

使用Maven时，可以使用自动扩展配置::

app.java.version=@java.version@

实用Gradle的语法如下::

app.name=${name}

HTTP压缩::

spring.compression.enabled=true

@WebFilter和@WebListener会自动注册。
