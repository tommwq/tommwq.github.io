gRPC简介

gRPC是一个基于HTTP2和protobuf的远程调用框架。gRPC支持多种语言，性能表现好。

定义接口

gRPC使用protobuf版本3定义消息和服务。关于消息的定义可以参考protobuf文档，这里介绍在gPRC中如何定义服务。在gRPC中，服务是一组接口的集合。服务具有唯一的名字。下面是一个不包含任何接口的服务定义:: 

service MyService {
}

接口定义在服务的上下文中，如:: 

service MyService2 {
    rpc MyInterface(InputType) returns (OutputType) {}
}

上面定义了服务MyServer2，并在这个服务中定义了接口MyInterface。接口MyInterface接受一个InputType类型的对象作为输入，产生一个OutputType类型的对象作为应答。InputType和OutputType必须是已经定义的protobuf消息。在接口MyInterface中，请求和应答是1-1对应的，这样的接口叫做请求应答接口。除此以外，gRPC还支持的3类接口，分别叫做服务端流接口、客户端流接口和双向流接口。服务端流接口接受一个对象作为请求，产生一组对象作为应答，它可以定义为:: 

rpc MyServerSideStream(InputType) returns (stream OutputStreamElementType) {}

客户端流接口和服务端流接口正好相反:: 

rpc MyClientSideStream(stream InputStreamElementType) returns (OutputType) {}

双向流接口是二者的组合:: 

rpc MyBiDirectionalStream(stream InputStreamElementType) returns (stream OutputStreamElementType) {}

从上面的几个例子可以看到，服务名、接口名和类型名全部使用驼峰法命名，这是gRPC所要求的。如果使用默认的工程目录解结构，proto文件应当位于src/main/proto目录。

生成接口代码

在build.gradle中加入:: 

compile 'io.grpc:grpc-netty-shaded:1.21.0'
compile 'io.grpc:grpc-protobuf:1.21.0'
compile 'io.grpc:grpc-stub:1.21.0'

和:: 

plugins {
    // Apply the java plugin to add support for Java
    id 'java'

    // Apply the application plugin to add support for building an application
    id 'application'
    id 'com.google.protobuf' version "0.8.9"
}

protobuf {
  protoc {
    artifact = "com.google.protobuf:protoc:3.7.1"
  }
  plugins {
    grpc {
      artifact = 'io.grpc:protoc-gen-grpc-java:1.21.0'
    }
  }
  generateProtoTasks {
    all()*.plugins {
      grpc {}
    }
  }
}

将proto文件放置到src/main/proto目录下。在发送请求时，为每个请求建立一个Task类，Task类需要从 android.os.AsyncTask派生。并持有Activity的java.lang.ref.WeakReference引用。Task需要实现三个方法： 

@Override protected String doInBackgroun(T... params);
@Override protected void onPostExecute(T result);

在doInBackground中，Task连接服务器，发送请求，将应答作为返回值。在onPostExecute中，Task处理应答。 如果此时Activity的WeakReference可用（get()返回为null对象），根据应答更新Activity对象。 

package io.grpc.helloworldexample;

import android.app.Activity;
import android.content.Context;
import android.os.AsyncTask;
import android.os.Bundle;
import android.support.v7.app.AppCompatActivity;
import android.text.TextUtils;
import android.text.method.ScrollingMovementMethod;
import android.view.View;
import android.view.inputmethod.InputMethodManager;
import android.widget.Button;
import android.widget.EditText;
import android.widget.TextView;
import io.grpc.ManagedChannel;
import io.grpc.ManagedChannelBuilder;
import io.grpc.examples.helloworld.GreeterGrpc;
import io.grpc.examples.helloworld.HelloReply;
import io.grpc.examples.helloworld.HelloRequest;
import java.io.PrintWriter;
import java.io.StringWriter;
import java.lang.ref.WeakReference;
import java.util.concurrent.TimeUnit;

public class HelloworldActivity extends AppCompatActivity {
  private Button sendButton;
  private EditText hostEdit;
  private EditText portEdit;
  private EditText messageEdit;
  private TextView resultText;

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_helloworld);
    sendButton = (Button) findViewById(R.id.send_button);
    hostEdit = (EditText) findViewById(R.id.host_edit_text);
    portEdit = (EditText) findViewById(R.id.port_edit_text);
    messageEdit = (EditText) findViewById(R.id.message_edit_text);
    resultText = (TextView) findViewById(R.id.grpc_response_text);
    resultText.setMovementMethod(new ScrollingMovementMethod());
  }

  public void sendMessage(View view) {
    ((InputMethodManager) getSystemService(Context.INPUT_METHOD_SERVICE))
        .hideSoftInputFromWindow(hostEdit.getWindowToken(), 0);
    sendButton.setEnabled(false);
    resultText.setText("");
    new GrpcTask(this)
        .execute(
            hostEdit.getText().toString(),
            messageEdit.getText().toString(),
            portEdit.getText().toString());
  }

  private static class GrpcTask extends AsyncTask<String, Void, String> {
    private final WeakReference<Activity> activityReference;
    private ManagedChannel channel;

    private GrpcTask(Activity activity) {
      this.activityReference = new WeakReference<Activity>(activity);
    }

    @Override
    protected String doInBackground(String... params) {
      String host = params[0];
      String message = params[1];
      String portStr = params[2];
      int port = TextUtils.isEmpty(portStr) ? 0 : Integer.valueOf(portStr);
      try {
        channel = ManagedChannelBuilder.forAddress(host, port).usePlaintext().build();
        GreeterGrpc.GreeterBlockingStub stub = GreeterGrpc.newBlockingStub(channel);
        HelloRequest request = HelloRequest.newBuilder().setName(message).build();
        HelloReply reply = stub.sayHello(request);
        return reply.getMessage();
      } catch (Exception e) {
        StringWriter sw = new StringWriter();
        PrintWriter pw = new PrintWriter(sw);
        e.printStackTrace(pw);
        pw.flush();
        return String.format("Failed... : %n%s", sw);
      }
    }

    @Override
    protected void onPostExecute(String result) {
      try {
        channel.shutdown().awaitTermination(1, TimeUnit.SECONDS);
      } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
      }
      Activity activity = activityReference.get();
      if (activity == null) {
        return;
      }
      TextView resultText = (TextView) activity.findViewById(R.id.grpc_response_text);
      Button sendButton = (Button) activity.findViewById(R.id.send_button);
      resultText.setText(result);
      sendButton.setEnabled(true);
    }
  }
}

fallback和intercepter

启动服务器时可以注册一个fallback HandlerRegistry。所有未匹配的请求会分派给fallback registry的lookupMethod(String name)方法。这个方法返回一个ServiceMethodDefinition，后者可以通过ServerServiceDefinition的方法getMethod(String name)得到。 ServerServiceDefinition可以通过BindableService.bindService()方法得到。gRPC编译器生产的XXXImplBase类就是一个BindableService。ImplBase虽然是一个抽象类，但实现了bindService()方法。

开发一个动态加载协议的代理。项目分为4个目录：

  • protocol。保存协议。
  • client。客户端。
  • server。服务器。提供4中类型的接口。
  • proxy。动态代理，同时提供mock server功能。 参数 -service-instance-table name=ip1,ip2 记录服务器示例地址。 参数 -mock-server=a.json 将文件内容作为应答返回。a.json包含：package、service、method_name、method_type、request_type、response_type、response_content。

公共参数：ApplicationRunner -server-address -server-port

gRPC在发送请求时会建立一个ClientCall<T,U>对象。Channel.newCall(MethodDescriptor, CallOptions)。

连接断开和心跳

如果没有调用onCompleted()而知直接关闭进程，对端会抛出io.grpc.StatusRuntimeException，提示“UNAVAILABLE io exception”，并调用onError()。对于双向流，必须先调用对端onCompleted()，然后调用自身的onCompleted()。否则调用channel.shutdown().awaitTermination()会超时。

当双向流一端失败后，另一端会立即调用onError(io.grpc.StatusRuntimeException)。因此一般情况下不需要心跳。
