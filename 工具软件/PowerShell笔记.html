变量作用域

PowerShell变量支持4中作用域，分别是$global、$private、$local和$script。 

$global:foo = 1
$script:bar = 2

哈希表

$h = @{"a" = "b"}

$h.Remove("c")
$h.Set_Item("a", "c")
Foreach ($key in $h.Keys)
{
    Write-Host $h[$key]
    Write-Host $h.$key
}

放置在函数前的注释，最后一行注释内容必须为空。示例:

#

调用dll

Function Init()
{
    $signature = @"
[DllImport("User32.dll")]
public static extern int MessageBox(int hWnd,string text,string title,int show);
"@

    $script:User32Dll = Add-Type -MemberDefinition $signature -PassThru -Name msgbox

    $signature = @"
[DllImport("Kernel32.dll")]
public static extern int FindFirstChangeNotification(
    string path,
    int recurse,
    int filter
);

public static extern int FindCloseChangeNotification(
    int handler
);
"@
    $script:Kernel32Dll = Add-Type -MemberDefinition $signature -PassThru -Name foobar2
}

Function MessageBox([string] $message, [string] $title)
{
    $script:User32Dll::MessageBox(0, $message, $title, 0)
}

在CentOS 7上安装PowerShell

$ curl https://packages.microsoft.com/config/rhel/7/prod.repo | sudo tee /etc/yum.repos.d/microsoft.repo
$ sudo yum makecache
$ sudo yum install powershell
$ pwsh

在Ubuntu 18.04上安装PowerShell

微软的软件仓库提供了Ubuntu版本的PowerShell安装包。导入微软软件仓库，就可以通过apt安装PowerShell了。 

curl https://packages.microsoft.com/keys/microsoft.asc | sudo apt-key add -
curl https://packages.microsoft.com/config/ubuntu/16.04/prod.list | sudo tee /etc/apt/sources.list.d/microsoft.list

导入后，更新apt并安装PowerShell。 

apt-get update
apt-get install -y apt-transport-https
apt-get update
apt-get install powershell

在Windows上通过PowerShell操作Linux

首先要在Linux上安装PowerShell、omi和psrp。微软仓库提供omi软件，因此添加仓库后可以直接安装 

# CentOS
yum install omi
# Ubuntu
apt install omi

安装完成后启动omi服务： 

/opt/omi/bin/omiserver -l

psrp需要手动安装。psrp的gitlab上提供了deb和rpm两个版本的二进制。psrp安装完毕后，还需要在Windows服务器上进行设置。 

winrm quickconfig REM Set-WSManQuickConfig
winrm set winrm/config/Client @{TrustedHosts="*"}

连接到远程Linux服务器 

$credential=Get-Credential
$option=New-PSSessionOption -SkipCACheck -SkipRevocationCheck -SkipCNCheck -SkipNetworkProfileCheck
Enter-PSSession -ComputerName IP -Credential $credential -Authentication basic -UseSSL -SessionOption $option
