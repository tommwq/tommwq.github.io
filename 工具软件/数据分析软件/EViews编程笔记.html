EViews编程语言

HelloWorld

EViews编程语言由EViews命令组成。简单的说，EViews代码就是由EViews命令组成的一个序列。下面是一个EViews程序的示例： 

'helloworld.prg
program helloworld

'draw a random line

wfcreate(wf=helloworld,page=dated) d 2018 2019

series r=@nrnd
show r.line

我们逐行介绍一下：

代码 说明
'helloworld.prg 以'开头的行是注释。
program helloworld 声明程序名为helloworld。
'draw a random line 也是注释。
wfcreate(wf=helloworld,page=dated) d 2018 2019 建立一个工作文件（WorkFile），时间从2018年到2019年，以天为单位。
series r=@nrnd 声明序列r，并使用随机数对r赋值。
show r.line 绘制r的曲线图。

可以看到，EViews程序是面向过程的。虽然EViews中存在对象的概念，但是这个对象不满足OOP特性要求。对象在EViews中的用途是保存数和结果。EViews源代码通常以“.prg”为扩展名。EViews不区分大小写。

运行

运行EViews.exe代码非常简单： 

eviews.exe helloworld.prg

概念

在EViews中有3个重要的概念：工作文件、页和对象。要理解EViews命令和程序，首先要理解这3个概念。

工作文件

工作文件（workfile）是一个文档，包含一个或多个页（page）。分析数据所用到的EViews对象就包含在页中。工作文件保含一个区间（Range），区间是工作文件使用的数据的范围。在分析时可以只选用区间的一个子集，叫做样本（sample）。工作文件中的页可以具有以下3中结构之一：

  • 带日期的（Dated），用于时间序列分析。
  • 不带日期的（Unstructured/undated），用于横截面分析。
  • 平衡面板（Balanced Panel），面板数据。

在工作文件中，每个页可以具有不同的结构。

命令wfcreate用于建立工作文件。下面的命令建立了一个工作文件，工作表的名字是monthly，工作表里包含一个页，页的名字是dated。工作表的区间从1950到2012，以月度为单位： 

wfcreate(wf=monthly, page=dated) m 1950 2012

这里的参数wf是工作文件名字，page是页名字。m定义了页的结构，表示按月份计算。1950和2012表示数据范围从1950（含）年到2012年（含）。

和m类似的参数还有：

类型 说明 附注
a 年 annual  
s 半年 semi-annual  
q 季度 quarterly  
m 月 montyly  
w 周 weekly  
d5 日（一周5日） 在某些早期版本中为“d”
d7 日（一周7日） 在某些早期版本中为“7”
u 不带日期 undated/unstructured  

下面的命令建立了名为undated的工作表，区间是1到50，包含一个名为state的页： 

wfcreate(wf=undated,page=state) u 50

下面的命令建立了平衡数据面板类型的页： 

wfcreate(wf=panel, page=country) q 1950 2010 100

区间后面的100表示横截面大小。

对于已经建立的工作文件，可以使用命令wfopen打开。wfopen也支持从其他应用中导入数据。下面的命令建立一个工作文件，并从Excel文件中导入数据： 

wfopen(page=panel) "c:\panel.xls"

在导入数据时，对于“表格”类型的数据（如Excel工作表），EViews认为数据按列排列。如果数据时按行排列，需要增加byrow参数： 

wfopen("page=transpose) "c:\transpose.xls" byrow

EViews支持的文件有Excel、HTML、CSV、ASCII、RATS、Stata、SPSS、SAS等。

页

页（page）可以理解为一个名字空间，EViews对象保存在页中。

命令pagecreate可以建立页，并将新页设置为当前页。它有5种形式，我们分别来介绍。 下面两种形式用于建立regular frequency或unstructured页。 

pagecreate(OPTIONS) freq[(SUBPERIOD_OPTS)] START_DATE END_DATE [NUM_CROSS_SECTIONS]
pagecreate(OPTIONS) u NUM_OBSERVATIONS

下面的形式可以基于其他页的对象建立新页。 

pagecreate(ID_METHOD[,OPTIONS]) IDENTIFIER_LIST [@srcpage PAGE_LIST]

IDENTIFIER_LIST 是序列编号列表。ID_METHOD可以是

id 使用观察值
idunion 使用观察值并集
idintersect 使用观察值交集

下面的形式基于其他页的对象建立横截面数据。 

pagecreate(idcross[,OPTIONS]) ID1 ID2 [@srcpage PAGE1 PAGE2]
pagecreate(idcross[,OPTIONS]) ID1 @range(FREQ, START_DATE, END_DATE) [@srcpage PAGE1]

每个新建的页都会默认包含两个对象：c和resid。这两个对象用于进行线性回归。在执行公式对象的ls命令后，c中第一个单元格（R1）记录了alpha值，第二个单元格（R2）记录了beta值。resid记录了残差项。

命令

EViews程序由命令组成。在EViews中，命令分为辅助命令和对象命令。我们前面见到的大多是辅助命令。辅助命令的用途是执行管理工作文件和页等与数据分析无关的操作。对象命令是通过对象调用的命令，比如程序helloworld中的： 

show r.line

就使用了对象命令r.line。使用EViews分析和预测数据，通常需要使用对象命令。要了解对象命令，首先要了解对象。

在使用对象前必须声明它。下面是一个例子，声明了一个名字是x的序列对象： 

Series x

对象声明的通用语法是： 

object_type(options) object_name

对象使用等号进行赋值，等号右边可以是函数或公式： 

Series x
Series y
x = @nrnd
y = x + 1

在上面的程序声明了两个序列对象x和y，序列长度由页的区间决定。x使用函数@nrnd赋值，x中的每一项被赋值为一个服从标准正态分布的随机数。y的每一项被赋值为对应x项的值加1。在赋值之前，序列中各项的值是NA。

声明了对象之后，就可以执行对象命令了： 

shwo x.line

上面是一个简单的对象命令，更完整的形式如下： 

action(action_opt) object_name.view_or_proc(options_list) arg_list

在这里，action是以下4个动词之一

do 执行命令
freeze 保存对象状态
print 显示
show 显示

object_name 是对象的名字， view_or_proc 是命令的名字，options_list是选修列表，arg_list是参数列表。我们看几个例子： 

Series x
x = @nrnd
show x.line

上面的程序中，调用了序列对象x的line命令，选项和参数为空。 

Series x
Series y
' x = ...
' y = '''
Equation e
e.ls y c x

上面的程序中，调用了公式对象e的ls命令，参数是y、c和x。

在使用对象命令时，动词是可以省略的，EViews会根据命令自动选择show或do作为动词，比如： 

gdp.line                 '等效于 show gdp.line
eq1.ls cons c gdp        '等效于 do eq1.ls cons c gdp
show eq1.ls cons c gdp   '等效于 先 do eq1.ls cons c gdp 再 show eq1

此外，对象的声明和命令也可以合并在一起： 

series x=@rnd
series y = log(x)
equation e.ls y c x

公式

公式（equation）是一个用于估算和预测的对象。定义了变量和解释变量的关系。

公式可以通过最小二乘法拟合得到： 

eq_name.ls(options) y x1 [x2 x3 ...]
eq_name.ls(options) specification

在第一个形式中，y是独立变量，x1、x2等是解释变量。如果需要增加截距项，可以使用常量c： 

eq1.ls y c x

第二个形式可以拟合复杂函数，如： 

eq1.ls q = beta(1) + beta(2)*(1^c(1) + k^(1-c(1)))

预测 

eq_name.forecast(options) yhat [y_se]
eq_name.forecast(options) yhat [y_se y_var]

拟合

假设有一个时间序列 vt ，并且根据分析可以确定， vt 满足下列等式

log(vt) = α + βt + epsilont

我们来拟合 α 和 β ，步骤如下：

  • 导入序列v。
  • 建立logvol和t对象。
series logv = log(v)
series t=@trend
  • 组合logv和t。
group g t logvol
  • 拟合。

选择对象g，点击“View”菜单，“Graph…”菜单项，将“Graph Type”选择为“Scatter”，将“Regression line”选择为“Fit lines”，点击“ok”按钮。

类似的，对于模型

log(vt) = α + β1 × t + β2 × t2 + εt

可以用下列的Eviews命令拟合： 

ls log(v) c @trend @trend^2

Jarque-Bera检验

Jarque-Bera检验是对样本数据是否具有符合正态分布的偏度和峰度的拟合优度的检验。Jarque-Bera检验不能用于小样本（小于30）检验。 

[h, p, JBSTAT,CV] = jbtest(x, alpha)
# x：样本，alpha：显著水平
# h：0表示接受x服从正态分布的假设，1表示不接受。p：检验值。JBSTAT：检验统计量。CV：临界值。

单位根检验（unit root test）

单位根检验用于检验序列中是否存在单位根。如果存在单位根，是非平稳序列。如果自回归的滞后系数为1，即

xt = xt-1 + ε

就称为单位根。单位根残差序列的误差不会随着样本数（时期数）的增加而衰减，模型中的差离是永久的。因此不能使用回归。对时间序列建模时，首先进行单位根检验。另一个角度看，对于一个差分方程，如果其特征方程的一个根是1，则称其为单位根。
