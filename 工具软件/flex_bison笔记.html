介绍Flex和Bison

Flex和Bison是用于处理结构化输入的软件，最早是为了开发编译器而开发的。编译过程可以分为两个步骤：词法分析（lexical analysis，也叫lexing或scanning）和语法分析（syntax analysis，也叫parsing）。词法分析将字符串分隔若干个token，语法分析处理各个token之间的关系。考虑下面的例子： 

alpha = beta + gamma ;

词法分析输出token序列：alpha，alpha，sign，beta，plus，sign，gamma，semicolon。语法分析判定“beta + gamma”是一个表达式，表达式的值被赋给alpha。

flex程序包含一组正则表达式。flex生成的词法分析程序在匹配到正则表达式后，会执行相对应的动作。下面是一个flex程序示例： 

/* just like Unix wc */
%{
int chars = 0;
int words = 0;
int lines = 0;
%}

%%

[a-zA-Z]+ { words++; chars += strlen(yytext); }
\n        { chars++; lines++ }
.         { chars++; }

%%

main(int argc, char **argv)
{
  yylex();
  printf("%8d%8d%8d\n", lines, words, chars);
}

这段flex代码被符号“%%”分隔为三个部分，第一部分是C语言的变量声明，第二部分是flex用于生成代码的正则表达式和动作，第三部分是C代码，将被复制到生成的源代码文件中。这段代码使用了一个变量yytext和一个函数yylex()。yytext表示被匹配正则表达式的字符串，yylex()则是flex库函数。现在尝试编译运行这段代码： 

$ flex fb1-1.l
$ cc lex.yy.c -lfl
$ ./a.out
The boy stood on the buring deck
shelling peanuts by the peck
^D
2 12 63
$

下面我们用一个计算器的例子来展示flex和bison如何一起工作。 

%%
"+"    { printf("PLUS\n"); }
"-"    { printf("MINUS\n"); }
"*"    { printf("TIMES\n"); }
"/"    { printf("DIVIDE\n"); }
"|"    { printf("ABS\n"); }
[0-9]+ { printf("NUMBER %s\n", yytext); }
\n     { printf("NEWLINE\n"); }
[ \t]  { }
.      { printf("Mystery character %s\n", yytext); }
%%

这段flex程序的第三部分没有C代码。flex库内置了一个main函数来调用scanner，所以可以省去第三部分代码。 

$ flex fb1-3.l
$ cc lex.yy.c -lfl
$ ./a.out
12+34
NUMBER 12
PLUS
NUMBER 34
NEWLINE
^D

函数yylex()是一个协程（coroutine），它会记录自己的内部状态，每次调用返回下一个token。flex生成的token实际包含两部分：文本和值。文本保存在yytext中，值保存在yylval中。我们可以手动为token定义值： 

%{
  enum yytokentype {
    NUMBER = 258,
    ADD = 259,
    SUB = 260,
    MUL = 261,
    DIV = 262,
    ABS = 263,
    EOL = 264
  };

  int yylval;
%}

%%
"+"    { return PLUS; }
"-"    { return MINUS; }
"*"    { return TIMES; }
"/"    { return DIVIDE; }
"|"    { return ABS; }
[0-9]+ { yylval = atoi(yytext); return NUMBER; }
\n     { return NEWLINE; }
[ \t]  { }
.      { printf("Mystery character %c\n", *yytext); }
%%

main(int argc, char **argv) {
  int tok;

  while (tok = yylex()) {
    printf("%d", tok);
    if (tok == NUMBER) printf(" = %d\n", yylval);
    else printf("\n");
  }
}

parser使用的语法和scanner不同。parser使用BNF（Backus-Naur Form）语法： 

<exp> ::= <factor>
      | <exp> + <factor>
<factor> ::= NUMBER
      | <factor> * NUMBER

下面是一个Bison程序： 

%{
#include <stdio.h>
%}

%token NUMBER
%token ADD SUB MUL DIV ABS
%token EOL

%%

calclist:
  | calclist exp EOL { printf("= %d\n", $1); }
  ;

exp: factor 
  | exp ADD factor { $$ = $1 + $3; }
  | exp SUB factor { $$ = $1 - $3; }
  ;

factor: term
  | factor MUL term { $$ = $1 * $3 }
  | factor DIV term { $$ = $1 / $3 }
  ;

term: NUMBER
  | ABS term { $$ = $2 >= 0? $2 : - $2; }
  ;

%%

main(int argc, char **argv) {
  yyparse();
}

yyerror(char *s) {
  fprintf(stderr, "error: %s\n", s);
}

bison程序的结构和flex程序一致。第一部分的%token NUMBER告诉bison这是一个token。$$表示目标符号，$1、$2等是冒号（“：”）右边的参数。 

$ bison -d fb1-5.y  # 生成fb1-5.tab.c fb1-5.tab.h
$ flex fb1-5.l
$ cc -o fb1-5.tab.c lex.yy.c -lfl

支持括号： 

%token OP CP

term: NUMBER
  | ABS term ...
  | OP exp CP { $$ = $2 }
  ;

"(" { return OP; }
")" { return CP; }
"//".* /* ignore comments */

（试验）支持十六进制 

0x[a-f0-9]+ { yylval = strtol(yytext); return HEXNUMBER; }
term | HEXNUMBER

使用flex

flex使用的正则表达式主要是POSIX扩展正则表达式。其中的特殊字符有：

. 匹配单个字符（除"\n"）。
[] 字符类。匹配方括号中出现的任一字符。如果首个字符是“^”，匹配不属于该字符类的字符。在字符类中可以使用连字符表示区间，如[0-9]表示[0123456789]。
[a-z]{-}[jv] 字符类。匹配到第一个字符类后，将输出后面字符类中的字符。
\^ 匹配行首。
\$ 匹配行尾。
{} 重复次数。如0{5}、A{1,3}。
\\ 转义。
\* 匹配0个或多个。
+ 匹配1个或多个。
? 匹配0个或1个。
| 备择匹配。
"" 匹配字符串。
() 匹配组。
\/ 上下文。0/1匹配01，但不匹配02。

在词法分析时经常遇到二义性。flex解决二义性的原则是：

  • 尽可能匹配最长的字符串。
  • 发生冲突时，按照规则在程序中出现的顺序作为优先级。
%option noyywrap
%{
  int chars = 0;
  int words = 0;
  int lines = 0;
%}

%%
[a-zA-Z]+ { words++; chars += strlen(yytext); }
\n        { chars++; lines++; }
.         { chars++; }
%%

main(int argc, char **argv) {
    if (argc > 1) {
        if (!(yyin = fopen(argv[1], "r"))) {
            perror(argv[1]);
            return 1;
        }
    }

    yylex();
    printf("%8d%8d%8d\n", lines, words, chars);
}

函数yylex()从文件（FILE）yyin中读取输入。当读取到yyin尾端后，yylex()会调用yywrap()函数。如果yywrap()返回1，yylex()就结束扫描。 如果yywrap()返回1，这表明输入了多个文件，并且yyin已经重新打开了。yylex()会继续从yyin中读取输入。
