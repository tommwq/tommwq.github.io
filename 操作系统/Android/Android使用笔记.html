开发Android程序中最常用到的类有：Activity、Service、BroadcastReceiver、ContentProvider和Intent。 Activity是为用户操作而展示的可视化界面。可视内容由一系列View对象按照一定层次组成。 View对象通过Activity的setContentView()方法放置到Activity中。 如果在程序中需要执行一些操作，操作不需要展示任何可视内容，应当将使用服务来执行这些操作。 为了能够控制操作行为，以及得到操作的结果，服务可以绑定到Activity上。 在状态发生变化的时候，系统会发送广播来通知Android程序。要接收这些广播，需要编写BroadcastReceiver。 Android程序也可以发布广播给自己或其他Android程序。BroadcastReceiver也没有可视组件。 如果在收到广播后需要通知用户，可以使用NotificationManager弹出消息栏。 Android系统内部使用SQLite数据库保存了一些数据，为了让应用程序可以访问这些数据，同时提供向后兼容的能力， Android系统提供了ContentProvider，用来访问系统内置的SQLite数据库。 Android中的广播通过Intent对象传递。Intent的用途就是在系统、Activity、Service、BroadcastReceiver之间传递数据。

在Android程序中使用上述组件时，首先要将组件注册到系统，注册的工具是AndroidManifest.xml。下面是一个例子： 

<?xml version="1.0" encoding="utf-8"?>
<manifest>
        <application>
                <activity
                        android:name=".activity.SplashActivity"
                        android:screenOrientation="portrait">
                        <intent-filter>
                                <action android:name="android.intent.action.MAIN"/>
                                <category android:name="android.intent.category.LAUNCHER"/>
                        </intent-filter>
                </activity>
                <service />
                <receiver />
                <provider />
        </application>
</manifest>

当应用程序开始运行第一个组件时，Android会这个程序启动一个Linux进程，所有组件都在这个进程中运行。

aapt将编译后的java类和资源文件打包到apk中。 每个Android程序运行在独立的虚拟机中。

资源文件 **

Android的资源文件包括字符串strings、颜色colors、数组arrays、动画anim、布局layout、图像drawable和音视频media等。 这些资源文件位于Android工程的res目录下。

资源文件目录布局 **

目录或文件 说明 /res/drawable 图形 /res/layout 布局 /res/values 字符串、颜色等 /res/values/strings.xml 字符串资源文件 /res/values/arrays.xml 数组资源文件 /res/values/colors.xml 颜色资源文件 res/values/dimens.xml 尺寸资源文件 /res/values/drawable 位图图像 res/anim 动画 /res/xml XML文件 /res/raw 原始文件

布局 ** 在布局xml中可以引入子布局xml，方法是： ::

<include layout="@layout/sublayout" />

新建项目建议使用android.support.design.widget.CoordinatorLayout作为最外层布局元素。

需要移动按钮时可以使用android.support.design.widget.FloatingActionButton。

其他 *

FloatingActionButton是悬浮于布局之上的一个按钮。

Snackbar是位于所有布局顶层的，全局唯一的，用于显示通知的控件。用法与Toast类似。

Activity应该从android.support.v7.app.AppCompatActivity继承。

底层导航栏要使用android.support.design.widget.BottomNavigationView，在其app:menu属性中设置导航菜单，如： ::

<!– 布局xml片段 –> <android.support.design.widget.BottomNavigationView android:id="@+id/navigation" android:layoutwidth="matchparent" android:layoutheight="wrapcontent" android:layoutgravity="bottom" android:background="?android:attr/windowBackground" app:menu="@menu/navigation"/>

<!– res/menu/nav.xml –> <?xml version="1.0" encoding="utf-8"?> <menu xmlns:android="http://schemas.android.com/apk/res/android">

<item android:id="@+id/navigationhome" android:icon="@drawable/ichomeblack24dp" android:title="@string/titlehome"/>

<item android:id="@+id/navigationdashboard" android:icon="@drawable/icdashboardblack24dp" android:title="@string/titledashboard"/>

<item android:id="@+id/navigationnotifications" android:icon="@drawable/icnotificationsblack24dp" android:title="@string/titlenotifications"/>

</menu>

然后设置setOnNavigationItemSelectedListener来处理导航。

使用配置的应用应该使用android.preference.PreferenceActivity。

读取资源时使用getResources.getString(int)。在kotlin中可以使用resources.getString(int)。

国际化 **

字符串国际化 ============= 在res目录下，如果为values目录添加后缀，可以实现国际化。如： ::

res/values-en-rUS 英语（美国） res/values-zh 中文 res/values-zh-cn 中文（简体）

图片国际化 ========== ::

drawable-en-hdpi drawable-zh-hdpi

常见语言编号 =========== ::

zh-rCN 中文（中国大陆） zh-rTW 中文（中国台湾） zh-rHK 中文（中国香港） en-rUS 英文（美国） en-rGB 英文（英国） en-rCA 英文（加拿大） en-rIE 英文（爱尔兰） en-rIN 英文（印度） en-rNZ 新西兰 en-rSG 新加坡 en-rZA 南非

使用自定义XML文件 *

  1. 将自定义XML文件放在res/xml目录下。如果没有xml目录，需要手动创建。假设自定义XML文件名字为myres.xml。
  2. 通过Context.getResources().getXml(R.xml.myres)获得自定义XML文件的XmlResourceParser的对象。
  3. 通过XmlResourceParser访问自定义XML资源。

::

/* <myresources> <message id="1">Hello</message> </myresources> */

XmlResourceParser parser = getResources().getXml(R.id.myres); try { for (; parser.getEventType() != XmlResourceParser.ENDDOCUMENT; parser.next()) { if (parser.getEventType() != XmlResourceParser.STARTTAG) { continue; }

String tagName = parser.getName(); if (tagName.equals("message")) { String id = parser.getAttributeValue(null, "id"); String content = parser.nextText(); // … } } } catch (XmlPullParserException e) { // … } catch (IOException e) { // … }
