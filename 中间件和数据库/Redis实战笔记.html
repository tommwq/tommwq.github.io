redis实战

Redis有5中数据类型：

  • 字符串（String）
  • 列表（List）
  • 集合（Set）
  • 散列表（Hash）
  • 有序集合（ZSet）

Redis自身就是一个键值对数据库，每个键都是字符串，值也可以是字符串。String提供了3个最基本的命令

set 设置值
get 读取值
del 删除键

处理最基础的字符串，Redis还提供了列表数据结构。列表是一个链表，它支持以下命令：

rpush 向列表右端插入元素
lrange 查看子列表
lindex 按下表读取元素
lpop 从左端弹出元素

列表的元素是有序的，可重复的。如果不要求元素无序，但要求元素唯一，可以使用集合。集合使用散列表实现。实际上，集合就是值恒为空的散列表。集合支持以下命令

sadd 添加元素
smembers 查看元素
sismember 检查元素
srem 移除元素

散列是一个关联数组，它支持以下操作

hset 设置键值
hget 读取值
hgetall 获取全部键值
hdel 删除键

有序集合是一种结合了优先队列和集合特点的数据结构。每个元素被赋予一个评分，元素按照评分从低到高排列。

zadd 添加元素
zrange 查看子集合
zrangebyscore 返回子集合及评分
zrem 删除元素

实例1：对文章进行投票 建立两个有序集合对文章进行索引。一个以发布时间为评分，一个以用户投票为评分。这样既可以按时间排序，也可以按质量排序。用户每投一票，修改有序集合中的评分。为了避免重复投票，可以建立一个集合保存已经投票的用户编号。为了避免投票集合膨胀，可以设置一个投票期。超过投票期后，不再允许用户投票，并删除投票集合。删除投票集合可以通过EXPIRES命令。

实例2：购物车 使用redis记录cookie，并为每个用户建立一个散列表作为购物车。

实例3：网页缓存 将网页渲染的结果保存在redis中作为网页缓存。

实例4：数据缓存。

Redis持久化配置： 

save 60 1000  # 快照
stop-writes-on-bgsave-error no
rdbcompression yes
dbfilename dumb.rdb

appendonly no
appendfsync everysec
no-appendsync-on-rewrite no
auto-aot-rewrite-percentage 100
auto-aof-rewrite-min-size 64mb

dir./

在以下条件下Redis回建立快照：

  • 客户端发送BGSAVE命令。
  • 客户端发送SAVE命令。收到SAVE命令后，redis在建立完快照之前不再响应任何命令。
  • 如果配置了save选型，当条件满足时，redis建立快照。
  • 客户端发送SHUTDOWN命令时。
  • Redis收到TERM信号时会执行SAVE命令，然后关闭服务器。
  • Redis受到另外一台Redis发送的SYNC命令，如果此Redis没有执行BGSAVE，这个服务器会执行BGSAVE。

dogpile效应指执行事务所需要的时间越长，就会有越多的待处理事务堆叠，这增加了单个事务执行所需的时间，并使得有超时限制的事务失败率上升。

使用Redis可以构建信号量。每个进程生成一个ID放到有序集合中，评分是请求获得信号量的时间戳。如果进程发现自己的排名小于信号量数量，进程得到信号量。否则进程将自己从有序集合中移除。

使用Redis的集合可以方便的建立搜索引擎。首先将原文中提取单词，移除助词，然后生成倒查表。在查询语句时，根据语句中的单词查找倒查表，计算交集作为结果。
