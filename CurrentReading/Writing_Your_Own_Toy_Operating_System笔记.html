计算机复位后首先执行POST（power-on self test）。然后根据CMOS记录的启动顺序，依次从启动设备中加载第一个扇区（512字节）到0:0x7c00位置。然后根据结尾是否是0xaa55来判断这个扇区是否是启动扇区。找到启动扇区后，计算机将设备号保存到寄存器DL，跳转到0:0x7c00，把控制权交给启动扇区。通常启动扇区需要执行以下操作：

  • 重置磁盘。
  • 在磁盘中找到内核，将其加载到内存中。
  • 启用A20跳线。
  • 设置IDT和GDT。
  • 进入保护模式。
  • 清理CPU指令缓存。
  • 设置386寄存器，包括ds、es、gs、fs、ss和esp。
  • 将控制权交给内核。

descriptor有64位，结构如下

0-15 limit 0-15
16-31 base 0-15
32-39 base 16-23
40-47 access byte
48-51 limit 16-19
52-55 flags
56-63 base 24-31

access byte的各位如下

0 Ac accessed 是否访问过
1 RW writable 是否可写
2 DC conforming 1 向下增长 0 向上增长
3 Ex executable 是否可执行
4 1  
5-6 Privl 特权等级
7 Pr 是否在内存中

flags有4位：

0    
1    
2 Sz 0 16位 1 32位
3 Gr granularity 0 bytes 1 KB

简单的操作系统只需要3个descriptor，一个空descriptor，一个代码descriptor，一个数据descriptor。代码descriptor的数据如下：

base address 0x00
limit 0xffff
present 1
privilege level 0
executable 1
direction 0
readable 1
granularity 1
size 1
access byte 0x9a
flags 0xcf

数据descriptor的数据如下

access byte 0x92
flags 0xcf

加载gdt 

lgdt [gdt pointer]

中断描述符占8字节，结构如下：

偏移 长度（字节） 名称 描述
0 2 offset_low 中断处理函数偏移量低地址
2 2 selector 代码段选择子
4 1 0 未使用
5 1 flags 属性
6 2 offset_high 中断处理函数偏移量高地址

flags的各位如下：

位 代码 描述
0..3 Gate type IDT gate type
4 storage segment 必须设置为0
5..6 privilege level  
7 present  

系统支持256中断，每个中断描述符占8字节，因此IDT表占2048个字节。 

lidt idt

idt的结构也是6字节，低2位是IDT表的字节数，高4位是IDT表线性地址。
