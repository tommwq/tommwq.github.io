Android系统概览

应用基础：Activity和Intent

Android应用包含4个基础的组件，分别是Activity、Service、BroadcastReceiver和ContentProvider。 Activity负责和用户交互，Service用于执行后台任务，BroadcastReceiver用于接收系统消息，ContentProvider用于存取数据。 除了ContentProvider外，基础组件可以用Intent启动。Intent可以包含一组（Bundle）描述组件的信息。

在编译期，Android会生成一个R.java文件，包含对资源的引用。资源、Java引用和XML引用的对照表如下：

资源 Java引用 XML引用
res/layout/main.xml R.layout.main @layout/main
res/drawable-hdpi/icon.png R.drawable.icon @drawable/icon
@+id/home_button R.id.home_buttom @id/home_button
<string name="hello"> R.string.hello @string/hello

Activity可以被切换到后台并重新启动。因此设备可能存在一个Activity的多个实例。 如果希望只保留一个Activity实例，可以在清单文件里说明。 方法是找到一个启动Activity，在其对应标签中添加属性： 

android:launchMode="singleInstance"

如果一个Activity拥有action为MAIN，category为LAUNCHER的<intent-filter>自标签， 这个Activity就是启动Activity。singleInstance保证了在一个Task中，Activity实例是唯一的。

如果希望Activity实例是全局唯一的，即无论那个程序启动了这个Activity，都返回同一个实例， 那么需要将launchMode设置为singleTask。 这种场景通常发生在Activity作为库被多个程序引用的情况下。

在设备屏幕旋转时，Activity会被销毁并重建。如果希望Activity保持同一种屏幕方向，可以设置 android:screenOrientation属性，值可取portrait或landscape。 实际上，屏幕旋转是配置变更的一个特例。当配置发生变更时，Activity被销毁被重建。 当软键盘弹出时，系统也认为配置发生了变更。 如果不希望系统自动重建Activity，可以设置android:configChanges属性。 configChanges的值是一组用竖线“|”分割的字符串，告诉系统当这些配置发生变更时不要重建 Activity。一些软件将这个属性设置为 

android:configChanges="orientation|keyboardHidden"

表示当屏幕发生旋转或软键盘弹出/隐藏时，不要重建Activity。

如果没有组织系统重建Activity，必须注意保留Activity信息，否则重建后信息会丢失，影响软件的体验。 Activity提供了两个函数完成信息保存和恢复工作： 

String userName = "Euler";

@Override
protected void onSaveInstanceState(Bundle state) {
        state.putString("name", userName);
}

@Override
protected void onRestoreInstanceState(Bundle state) {
        userName = state.getString("name");
}

方法onPause()和onResume()也可以完成类似的工作。不过出于语义明确的角度，不建议使用它们。

从一个Activity中启动另一个Activity可以用方法startActivity： 

Intent intent = new Intent(this, NextActivity.class);
startActivity(intent);

startActivity是从Context继承的，因此上面的点也可以使用在Context上。要关闭一个Activity， 可以调用finish()方法。

启动另一个Activity还可以用startActivityForResult方法。 这种情况用于新启动的Activity负责进行一些计算，并将结果返回原Activity。 

Intent intent = new Intent();
intent.putExtra(KEY, "some value");
startActivityForRequest(intent, requestCode);

计算结果通过onActivityResult接收 

@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    updateUI();
    super.onActivityResult(requestCode, resultCode, data);
}

Intent filter可以是action、数据或category，最常用的是ACTION_VIEW。

在Activity之间传递数据

在ActivityA中调用startActivityForResult(intent, RequestCode)启动ActivityB。 参数通过intent传递。在ActivityB中，通过getIntent()得到数据，进行计算，将结果通过setResult(RESULT_OK, intent)保存，然后通过finish()返回ActivityA。返回后，ActivityA.onActivityResult(RequestCode, ResultCode, intent)被调用，ActivtyA通过intent得到结果。

倒计时任务可以使用android.os.CountDownTimer完成。CountDownTimer提供了onTick和onFinish方法，分别在每次计时和计时结束时调用。 

new CountDownTimer(totalMilliseconds, tickMilliseconds) {
    public void onTick(long remainMillis) {
        timeLabel.setText("remaining seconds: " + remainMillis / 1000);
    }
    public void onFinish() {
        timeLabel.setText("done");
    }
};

向一个Activity发送消息可以通过android.os.Handler完成。在Activity中建立一个Handler成员，重载void handleMessage(Message)函数，就可以处理自定义消息。handleMessage在主线程中工作，因此，消息处理过程可以更新界面，但不应当耗时太长。

Service需要注册到清单文件中，在<application>标签下添加子标签<service>。启动Service和关闭Service都通过Intent进行： 

Context.startService(new Intent(this, MyService.class));
Context.stopService(new Intent(this, MyService.class));

创建Widget的方法 

import android.appwidget.AppWidgetManager;
import android.appwidget.AppWidgetProvider;
import android.content.Context;
import android.widget.RemoteViews;

public class MyWidgetProvider extends AppWidgetProvider {

        public void onUpdate(Context context, AppWidgetManager manager, int[] widgetIds) {
                super.onUpdate(context, manager, widgetIds);

                for (int i = 0; i < widgetIds.length; i++) {
                        updateWidget(context, manager, widgetIds[i]);
                }
        }

        public void updateWidget(Context context, AppWidgetManager manager, int widgetId) {
                RemoteViews views =new RemoteViews(context.getPackageName(), R.layout_widget_layout);
                manager.updateWidget(widgetId, views);
        }
}
