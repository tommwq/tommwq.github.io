服务启动流程。

调用ZebraRun.run()启动程序。 查找@ZebraConf，设置配置中心位置。通过HTTP请求从配置中心拉取配置，写入本地目录。配置的注入见PropertiesContent类。 建立并运行SpringApplication。由于@ZebraService派生自@Service，因此Spring会为服务类建立bean。 GrpcServiceRunner作为CommandLineRunner以@Order(1)运行。GrpcServiceRunner扫描带有@ZebraService的bean，生成服务配置（RpcServiceBaseInfo）列表，通过RpcServiceExport发布（export方法）服务。 RpcServiceExport发布服务时，首先建立一个GrpcEngine对象，委托GrpcEngine建立一个io.grpc.Server对象，然后启动Server。 GrpcEngine通过NettyServerBuilder建立Server对象。首先加入MonitoringServerInterceptor，然后将服务加入Server，并在etcd3中注册服务。这个过程在方法getServer中实现。

调用服务流程。 调用ZebraRun.run()启动程序。 GrpcReferRunner作为InstanitationAwareBeanPostProcessorAdapter运行，注入@ZebraReference对象。 注入的对象是GrpcReference.getProxyObject方法返回的代理类。GrpcReference是RpcServiceExport的派生类。返回的代理类是通过RpcServiceExport.getGrpcEngine().getClient()得到的，是一个GrpcProtocolClient接口。具体实现是DefaultProxyClient。 DefaultProxyClient继承自GrpcProtocolClient，其中getGrpcClient为具体的接口类（根据类型名，通过反射加载）建立一个动态代理DefaultProxyClientInvocation。这个动态代理用于建立GrpcRequest对象。 DefaultProxyClientInvocation是AbstractClientInvocation的派生类，后者是实际的代理。并通过断路器，调用UnaryCallListner的run方法，发送消息。

整个Zebra在调用上的核心类是GrpcEngine。

Zebra实现泛化调用的方法。

当服务器程序启动时，首先对注解进行扫描，找到gRPC服务。假设这个服务名字是MyService。Zebra会为MyService生成一个新服务，名字是MyServiceJSON。新服务拥有相同的方法名，区别是输入和输出的参数不同： 

message JsonRequest {
 string message = 1;
 string reqType =2;
}

message JsonReply {
  string message = 1;
}

现在客户端通过GenericClient发送请求到MyService。Zebra会将请求号改写为MyServiceJSON，并将传入的参数格式化为JSON字符串。这一步是在客户端完成的。之后，当服务器收到MyServiceJSON请求后，将JSON字符串首先解析为JSONObject对象，然后调通过Pojo2Protobuf建立protobuf对象，接着调用实际的接口，将返回数据通过protobuf2pojo封装成Java对象，然序列化为JSON字符串。Java对象和Protobuf对象的转换借助了maven插件。在编译时，调用命令protoc –descriptorsetout，得到proto文件信息，并据此生成Pojo类和pojo与protobuf转换的代码。整个过程大致如下：

客户端通过GenericClient发起调用。 Zebra将请求改写为MyServiceJSON，将参数封装为JSON字符串。 服务器收到MyServiceJSON请求，将JSON字符串反序列化为Pojo对象，再通过编译期生成的Pojo2Protobuf方法，得到Protobuf对象。 服务器调用实际的接口，得到返回的Protobuf对象，将其按照相反的顺序，生成JSON字符串，返回给客户端。

Gateway网关也是通过返回调用来访问后端服务的。网关使用Vertx开发，对外部作为一个HTTP服务器。手机客户端发送的请求以HTTP协议传输，基础参数保存在url中，业务参数封装成JSON通过body传递。网关收到请求后，进行必要的安全检查，比如签名，请求号等，然后通过vertx内部的消息总线EventBus传递给TransmitVerticle对象，或者将HTTP请求转换为GerericClient的泛华调用。
