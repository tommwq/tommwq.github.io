ThreadLocal

ThreadLocal是线程本地对象，保存在Eden中的一个区域。使用ThreadLocal可以避免锁竞争，提高效率。

ThreadLocal的方法有： 

public T get();
protected T initialValue();
public void remote();
public void set(T value);

Java运行时数据

Java运行时数据分为6类： -程序计数器。这是线程私有的。 -Java虚拟机栈。也是线程私有的，存储局部变量、操作栈、方法返回值等。 -本地方法栈。和Java虚拟机栈类似。 -Java堆。多线程共享的。 -运行时常量池。记录类中的常量表。

Java GC

Java采用分代垃圾收集机制，这一机制的基础是若年代假设，即：

  • 大部分新对象很快变得不可达。
  • 老对象很少引用新对象。

基于这个假设，Java把堆内存划分为老年代和新生代。新生代包含Eden和2个Survivor区域。Eden保存新创建的对象。如果Eden填满，Java执行Minor GC，收集Eden和Survivor中的不可达对象，将仍然使用的对象保存在一个Survivor中。在Minor GC之后，Eden和一个Survivor被清空。在Minor GC时，如果一个对象在Survivor中经历了多次GC仍然存活，这个对象被移动到老年代。

HashMap和HashTable

HashMap不是线程安全的，HashTable是线程安全的。 HashMap运行key和value是null，HashTable不允许。 在单线程中HashMap比HashTable效率高。

HashMap实现机制

HashMap是一个散列表，元素保存在hash(key.hashcode)&(len-1)的桶中。每个桶以链表形式组织。
