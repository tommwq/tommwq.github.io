和软件界面（UI）相关的要素有：界面、数据、事件、业务。

MVC模式。在MVC模式中有三个要素：业务模型（Model）、视图（View）和控制器（Controller）。 在MVC模式中，视图将用户事件发送给控制器，控制器处理业务流程，并根据流程结果更新模型。模型更新后，控制器更新视图。使用伪代码描述如下： 

void View::on_user_action(Action action) {
  controller->on_user_action(action);
}

void Controller::on_user_action(Action action) {
  BusinessLogicRequest request = action.get_request();
  BusinessLogic logic = action.get_logic();
  BusinessLogicResult result = logic.do_business(request);

  model->update(result);
}

void Model::update(BusinessLogicResult result) {
  State new_state = next_state(current_state, result);
  controller->update_state(new_state);
}

void Controller::update_state(State new_state) {
  view->render(new_state);
}

MVP模式包含三个要素：模型（Model）、视图（View）和演示器（Presenter）。 视图将用户事件发送给演示器，演示器调用模型来执行业务逻辑，并将业务结果通知视图。

MVC模式和MVP模式都是通过引入中间层的方式，将视图和模型解耦。

MVVM模式包含三个要素：模型（Model）、视图模型（ViewModel）和视图（View）。 视图模型将模型和视图连接起来。在MVVM模式中，模型和视图模型，以及视图模型和视图之间，都采用双向绑定的关联方式。 这也是MVVM和MVC、MVP的不同之处。在MVC中，控制器和视图之间是两个单项关联。控制器和模型之间也是一样。 在MVP中，演示器和模型之间是单向联系。

在Android中应用MVC模式时，视图是布局XML文件，模型是实体（Entity）类，控制器是Activity业务逻辑。MVC模式会使得控制器代码膨胀和杂乱。

在Android中应用MVP模式时，视图是布局XML和Activity，负责页面渲染和用户交互。模型是实体类。
