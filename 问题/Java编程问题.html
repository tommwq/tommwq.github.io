找不到MySQL驱动

表现

程序使用Spring Data JPA，在应用启动时报告

2019-04-10 11:38:37.870 WARN 4344 — [enterApp.main()] com.zaxxer.hikari.util.DriverDataSource : Registered driver with driverClassName=com.mysql.jdbc.Driver was not found, trying direct instantiation.

原因

缺少mysql-connector-java包。

解决方法

增加mysql-connector-java依赖项。

Spring Boot创建bean的时机。

SpringApplication.run()方法建立了一个ConfigurableApplicationContext对象。对于Web程序，这个对象的实际类型是org.springframework.boot.web.servlet.context.AnnotationConfigServletWebServerApplicationContext。 这个Context对象执行prepareContext()方法，建立BeanDefinitionLoader，并委托后者加载Bean。这时，程序只加载一个Bean（？@SpringBootApplication）。

然后Context执行refresh()方法时，会对注解进行扫描，并生成Bean对象。refresh()执行的步骤包括

  • prepareRefresh() 设置placeholder值。
  • invokeBeanFactoryPostProcessors() 搜索并建立Bean对象。
    • PostProcessorRegistrationDelegate.invokeBeanDefinitionPostProcessors()
      • invokeBeanDefinitionRegistryPostProcessors()
        • ConfigurationClassPathPostProcessor.processConfigBeanDefinition()
          • ConfigurationClassPather.parse()
          • ConfigurationClassBeanDefinitionReader.loadBeanDefinitions()
        • AbstractAutowireCapableBeanFactory.getBean()
          • doGetBean()
            • DefaultSingletonBeanRegistry.getSingleton()
            • AbstractAutowireCapableBeanFactory.createBean()
              • doCreateBean()
              • createBeanInstance()
              • instantiateUsingFactoryMethod()
                • getBean() [optional]

在使用 MySQL 8.0 时重启应用后提示 com.mysql.jdbc.exceptions.jdbc4.MySQLNonTransientConnectionException: Public Key Retrieval is not allowed

最简单的解决方法是在连接后面添加 allowPublicKeyRetrieval=true

文档中(https://mysql-net.github.io/MySqlConnector/connection-options/)给出的解释是：

如果用户使用了 sha256password 认证，密码在传输过程中必须使用 TLS 协议保护，但是如果 RSA 公钥不可用，可以使用服务器提供的公钥；可以在连接中通过 ServerRSAPublicKeyFile 指定服务器的 RSA 公钥，或者AllowPublicKeyRetrieval=True参数以允许客户端从服务器获取公钥；但是需要注意的是 AllowPublicKeyRetrieval=True可能会导致恶意的代理通过中间人攻击(MITM)获取到明文密码，所以默认是关闭的，必须显式开启

AbstractBeanFactory在AbstractApplicationContext.finishBeanFactoryInitialization阶段被调用。 注入@Value的是AbstractBeanFactory.resolveEmbeddedValue()方法。会调用AbstractPropertyResolver.doResolvePlaceholders。这个会从propertySourceList成员中查找属性。 这里默认有两个PropertySource：environmentProperties和localProperties 可以通过调用BeanFactory.addEmbeddedValueResolver添加Resolver。

@FunctionalInterface public interface StringValueResolver { @Nullable String resolveStringValue(String var1); }

监听(不行） org.springframework.context.event.ApplicationContextEvent 得到context。 接口ConfigurableApplicationContext.getBeanFactory()

自定义BeanPostProcessor处理。如果是全局性的，自定义BeanFactoryPostProcessor 处理。BeanFactoryPostProcessro必须同时实现priorityorder，返回order1。否则添加的resolver顺序不对，会抛出异常。

AbstractHandlerMethodMapping.getDirectUrls()里的path可能为null。

AbstractHandlerMethodMapping.detectHandlerMethods() 查找handler方法。 initHandlerMethods调用detectHandlerMethods()。这个在afterPropertiesSet()中调用。 initHandlerMethods遍历Bean，委托RequestMappingHandlerMapping判断一个Bean是否是Handler。判断的标准是看类是否具有Controller/RequestMapping对象。
